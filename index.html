🧩 POC Tile Editor

A tile pipeline split into three decoupled roles, joined only by a bundle.

role ①

Editor — the tool

Where human intent goes in: paint, fill, rect, layers. Exports to Godot / Unity / Tiled / Generic.

→
role ②

Bundle — the contract

Data + resources only, no logic: map.json (tile-id grid) + tileset PNG(s). The seam between tool and game.

bundles/demo/
→
role ③

Game — the runtime

Consumes a bundle and nothing else. Owns collision, player, camera, skins. Press T for day/night.

Directory map

poc-tile-editor/
├── editor/          ① the tool      index.html · css/ · js/ (+ js/exporters/)
├── bundles/         ② the contract  demo/ → map.json + tileset.day.png + tileset.night.png
├── game-runtime/    ③ the game      index.html · game.js   (depends only on a bundle)
│
├── engine-templates/   Godot (.gd) & Unity (.cs) runtime importers
├── tools/              build scripts: make-demo-bundle · make-sample-tileset · make-tileset-variant
├── samples/            source tilesets
├── ARCHITECTURE.md     the three roles, the bundle contract, reskin rules
└── README.md
